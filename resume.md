### AI 검색엔진 SDK 개발 (2024/Q4)

> _React.js, TypeScript_

AI 검색엔진 SDK를 기획부터 설계·개발까지 전담했습니다. 그 결과 기존 고객사 웹사이트의 검색 실패율을 약 58% 줄이고, 클릭 및 구매 전환율을 약 50%까지 끌어올렸습니다.

**문제 정의**

- 기존에는 고객사 쇼핑몰 엔드유저의 구매 경험을 향상하고자 AI 챗봇 SDK 솔루션을 개발했음
- 고객사 유입 후 데이터 분석 결과, 챗봇 이용률이 저조해 재계약으로 이어지지 않음
- 문제 재정의를 위해 TF 팀에 참여해 직접 외부 클라이언트와 인터뷰를 진행한 결과, 웹빌더 고객사들이 검색 엔진 성능을 페인포인트로 가지고 있음을 확인함

**구현 사항**

- 2주 스프린트 내에 웹빌더 고객사의 검색 성능 이슈를 해결하는 AI 검색 엔진 SDK 개발

**개발 시 고민 사항**

- 자사 SDK는 UI도 제공하려고 했으나, 웹빌더 고객사는 자체 개발 인력이 존재 X
- 개발 리소스를 줄이고 다수 고객사를 확보하기 위해, 웹빌더 제공 옵션에 따라 검색창과 결과 UI를 손쉽게 커스터마이징할 수 있도록 SDK를 구현

**성과**

- 고객사 웹 사이트의 UI/UX를 해치지 않으면서 검색 엔진 효율을 크게 개선
- 검색 실패율 약 58% 감소 + 클릭율 / 구매 전환율 약 50% 증가

### 리테일 미디어 어드민 개발 및 유지보수 (2024/Q2)

> _Next.js, React.js, TypeScript_

초개인화 타겟팅 광고를 관리하는 리테일 미디어 어드민 프로젝트입니다. 캠페인 운영을 위한 어드민 대시보드와 고객사 웹에 광고·추천 위젯을 연동하는 로직을 설계·구현했습니다.

1. **Next.js 메이저 버전 점진적 업그레이드 (v12 → v15)**
   - **마이그레이션 계기**: 서비스 리뉴얼로 신규 페이지 구현이 많아졌고, App Router 도입 시 생산성이 향상되리라 판단해 점진적 전환 결정
   - **고민 사항**: 기존 페이지를 한 번에 리뉴얼하기에는 리소스가 부족했고, 기능 개발도 계속 병행해야 했음
   - **구현 방법**: Page Router와 App Router 앱을 각각 별도 도메인에 배포하고, 프록시를 통해 하나의 앱으로 통합해 점진적으로 마이그레이션
   - **성과**: App Router로 점진적 마이그레이션 해, 개발 생산성을 높이고 서비스 품질을 개선
2. **개인화 추천 상품 위젯 SDK 개발**
   - **문제 정의**: 기존에는 고객사 웹에 직접 스크립트를 삽입해 추천 상품 UI 구현 → 고객사 UI에 의존도가 높아 스타일 변경 시 레이아웃이 깨지고 유지보수가 어려웠음
   - **구현 내용**: 어드민에서 설정한 위젯 정보를 기반으로 스크립트를 생성하고, 고객사는 해당 스크립트를 자사 웹에 삽입하면 서버에서 추천 상품을 설정한 UI대로 렌더링하도록 SDK 구현
   - **성과**: 고객사 레이아웃 이슈 해소 및 추천 UI 연동에 필요한 리소스를 크게 절감

### 디자인 시스템 : 스토리북 빌드 속도 개선 (2024/Q1)

> _React.js, TypeScript, Storybook_

스토리북 빌드 성능 향상을 위해 번들러를 Webpack5에서 Vite로 마이그레이션했으며, 그 결과 개발 모드 빌드 시간이 약 90% 단축되었습니다.

**문제 정의**

- 스토리북 개발 모드 Warm Start time이 3s로 매우 느려 개발 생산성이 떨어져 빌드 속도 개선

**기술 선정 과정**

- 개발 모드 빌드 속도를 개선하기 위해, 기존 Webpack5에 esbuild-loader를 이용할지 Vite로 마이그레이션을 할지 고민
- 결론: Vite를 이용하는 것이 개발 모드 속도가 더 빠르리라 판단해 Vite로 결정

**트러블슈팅: 호환 이슈 발생**

- Vite로 마이그레이션 하는 과정에서 호환 이슈가 발생해 기존 기술 스택 변경
- 이슈 1) OpenSSL API 버전 이슈가 발생→ Storybook v7로 버전업
- 이슈 2) 팬텀 디팬던시로 인해 모듈 포맷 호환 이슈가 발생 → 패키지 매니저를 Yarn 1에서 pnpm으로 마이그레이션

**성과: 개발 모드 기준 빌드 시간 약 90% 감소**

- 14s → 2.5s로 개발 모드 Cold Start time 감소 (82% 감소)
- 3s → 50ms로 개발 모드 Warm Start time 속도 감소 (98% 감소)
